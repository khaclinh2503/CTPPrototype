## What Was Built

### ctp/skills/handlers_upgrade.py

3 skill handlers cho trigger `ON_UPGRADE`:

**handle_teddy (SK_TEDDY)**
- Effect 1: Set `tile.building_level = 5` (mien phi, player chi tra tien den `chosen_level`)
- Effect 2: `random.randint(0, 99) < 60` → di chuyen den Travel tile gan nhat
- D-50 cascade: Sau khi tao L5, goi `engine.fire("ON_UPGRADE", player, cascade_ctx)` voi `new_level=5` va guard `from_teddy=True` tranh vong lap
- D-20: Mutually exclusive voi SK_GAY_NHU_Y (enforce o cap assign, khong enforce trong handler)

**handle_o_ky_dieu (SK_O_KY_DIEU)**
- Guard: `new_level != 4` → return None (chi trigger tai L4, D-23)
- Return: `{"type": "sweep_walk", "steps": 32}` — FSM xu ly viec di chuyen thuc te
- T-02.5-08: Bounded 32 steps, khong di chuyen truc tiep trong handler

**handle_mong_ngua (SK_MONG_NGUA)**
- Guard: `new_level != 5` → return None (chi trigger tai L5, D-24)
- Tim Resort cung canh bang `board.get_row_non_corner_positions(tile.position)`
- Unowned → `resort.owner_id = player.player_id`, them vao `owned_properties`
- Opponent → `old_owner.remove_property()`, doi `owner_id`, them vao player (D-28: khong thay doi `building_level`)
- Own Resort → return None (no effect)
- No Resort on side → return None (fail silently)
