# Phase 02.5 Plan 03: MOVE-trigger handlers (SK_CAM_CO, SK_PHA_HUY) Summary

## One-liner

SK_CAM_CO claims unowned CITY tiles on pass with [1,3,5,7] decay; SK_PHA_HUY destroys opponent property and grants 50% invested cost refund; both restricted to dice_walk/sweep_walk per D-57.

## What Was Built

### `ctp/skills/handlers_move.py`

**handle_cam_co** (ON_MOVE_PASS):
- Only fires during `dice_walk` or `sweep_walk` (D-57)
- Claims unowned CITY tiles: `tile.owner_id = player.player_id`, `building_level` stays 0
- Uses `player.cam_co_current_rate` (initialized at turn start by caller)
- Rate decay per activation: `[1, 3, 5, 7]` sequence via `cam_co_decay_index`

**handle_pha_huy** (ON_MOVE_PASS):
- Only fires during `dice_walk` or `sweep_walk` (D-57)
- Destroys opponent-owned CITY/RESORT tile: `building_level=0`, `owner_id=None`
- Player receives `calc_invested_build_cost(board, pos) * 0.5`
- After destruction, chains to CamCo via `engine.fire("ON_MOVE_PASS", player, ctx)`

**check_traps** (helper):
- Returns `(should_stop: bool, trap_type: str | None)`
- `teleport` and `skill_walk` are immune (D-55)
- `dice_walk`, `sweep_walk`, `travel_walk` are susceptible
- Illusion does NOT trigger against its creator (`board.illusion_owner_id`)
- Both traps are consumed (set to `None`) after triggering

Both handlers registered in `SKILL_HANDLERS` dict at module import.

### `tests/test_handlers_move.py`

17 tests across 4 test classes:
- `TestHandleCamCo`: 5 tests (claim, owned-tile guard, decay, teleport-immune, sweep_walk)
- `TestHandlePhaHuy`: 3 tests (destroy L3, own-property guard, teleport-immune)
- `TestPhaHuyCamCoChain`: 2 tests (chain via engine mock, direct end-to-end)
- `TestCheckTraps`: 7 tests (illusion, stop_sign, teleport bypass, skill_walk bypass, creator immunity, no traps, travel_walk susceptible)

All 17 tests pass.

## Decisions Made

1. `check_traps` uses `getattr(board, "illusion_owner_id", None)` to safely handle Board instances that do not yet have `illusion_owner_id` defined — avoiding breaking existing tests.

2. `handle_cam_co` uses `player.cam_co_current_rate` directly rather than calling `engine.calc_rate()` — this is intentional because the rate already includes decay and is managed per-turn by the caller. The engine's rate check (before calling the handler) uses the base rate; the handler uses the decaying per-turn rate.

3. The PhaHuy->CamCo chain calls `engine.fire("ON_MOVE_PASS")` only when `engine is not None` and `"SK_CAM_CO" in player.skills`, avoiding infinite loops when engine is None (test context).
