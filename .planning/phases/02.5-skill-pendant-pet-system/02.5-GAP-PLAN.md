# Phase 02.5 GAP Plan — Toll Integration + Moonwalk Fix

## Goal

Fix 2 verification gaps:
1. **SK_MOONWALK backward**: `_pending_direction` consumed in `_do_move`
2. **Toll skill integration**: ON_LAND_OPPONENT/ON_OPPONENT_LAND_YOURS fire BEFORE `strategy.on_land()`, results injected into toll calculation

## Approach

**Toll restructure strategy**: Move skill hook firing to BEFORE on_land(), pass results as kwargs through to `apply_toll_modifiers()`. This avoids refactoring `strategy.on_land()` return values and keeps the existing D-44 toll modifier pipeline intact.

**Moonwalk**: Simple direction reversal in `_do_move()` — backward movement wraps position using `(old_pos - 1 - dice_total) % 32 + 1`.

## Task Order

GAP-T1 (Moonwalk) and GAP-T2+T3 (Toll) are independent. GAP-T4 (Tests) depends on both.
