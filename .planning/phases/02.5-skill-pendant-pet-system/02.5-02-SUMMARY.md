## What Was Built

### ctp/skills/handlers_roll.py

- `handle_xxct`: Returns `{"type": "dice_modifier", "options": [dice-1, dice, dice+1]}` with `min(dice-1, 1)=1` guard. Registered as `SKILL_HANDLERS["SK_XXCT"]`.
- `handle_xe_do`: Returns `{"type": "dice_replace", "options": <parity set>}`. Even dice → `[2,4,6,8,10,12]`, odd → `[1,3,5,7,9,11]`. Registered as `SKILL_HANDLERS["SK_XE_DO"]`.
- `handle_moonwalk`: Returns `{"type": "direction_choice", "options": ["forward", "backward"]}`. Registered as `SKILL_HANDLERS["SK_MOONWALK"]`.
- `resolve_roll_modifiers(results, dice_result, choose_fn)`: Merges active roll modifiers per D-46 — XeĐo overrides XXCT; Moonwalk appends backward direction. Returns `{"steps": [...], "directions": [...], "choose_fn": fn}` for FSM Plan 16 consumption.

### tests/test_handlers_roll.py

20 tests covering:
- XXCT: dice=5 → [4,5,6]; dice=2 → [1,2,3]; dice=1 min guard
- XE_DO: dice=4 (even) → [2,4,6,8,10,12]; dice=7 (odd) → [1,3,5,7,9,11]
- MOONWALK: forward+backward; exactly 2 options
- SKILL_HANDLERS registration for all 3 skills
- resolve_roll_modifiers: no modifiers, XXCT alone, Moonwalk alone
- XXCT+Moonwalk: 3 steps × 2 dirs = 6 combos
- XeĐo+Moonwalk: 6 steps × 2 dirs = 12 combos
- XeĐo overrides XXCT (order-independent)
- choose_fn passed through

**Test result: 20/20 passed.**

## Decisions Made

1. **XeĐo overrides XXCT in resolve_roll_modifiers**: Priority check — `if has_xe_do` before `elif has_xxct`. Order-independent by design. Matches D-46 GD notes.
2. **resolve_roll_modifiers returns dict, not final choice**: FSM (Plan 16) owns the actual player interaction/AI selection. Handler layer only builds the pool.
3. **Handlers registered at module level**: `SKILL_HANDLERS["SK_XXCT"] = handle_xxct` etc. at module scope, consistent with registry pattern in `ctp/skills/registry.py`.
