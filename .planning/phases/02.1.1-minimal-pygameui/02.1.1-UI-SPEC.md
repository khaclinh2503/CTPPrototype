# Phase 02.1.1 — UI Design Contract

> Visual and interaction contract for Phase 02.1.1: Minimal PygameUI.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Pygame 2.6.1 — direct drawing API) |
| Preset | not applicable |
| Component library | none — all surfaces drawn via `pygame.draw.*` |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Token offset when stacking multiple players on same tile |
| sm | 8px | Padding inside tile rhombus label, icon gaps |
| md | 16px | Padding inside panel sections |
| lg | 24px | Vertical gap between player info blocks |
| xl | 32px | Panel top/bottom margin |

Exceptions:
- Tile rhombus half-size `s ≈ 24px` (derived from diamond geometry: `R=280`, spacing `~49.5px` between centers — not a spacing token, it is a geometry constant)
- Touch targets: not applicable (keyboard-only control, no mouse interaction in this phase)

Source: RESEARCH.md (Pattern 3 — verified geometry)

## Typography

All text rendered with `pygame.font.SysFont(None, size)`. No external font files.

| Role | SysFont Size | Weight | Usage |
|------|-------------|--------|-------|
| Tile label | 14 | normal (400) | Position number + space type abbreviation inside each rhombus |
| Token | 16 | normal (400) | Player letter (A/B/C/D) rendered on tile |
| Panel body | 18 | normal (400) | Player name, cash, total_assets values, event log lines |
| Panel heading | 22 | normal (400) | Section headers: player names, speed indicator |

Note: `pygame.font.SysFont` does not expose CSS font weights. "normal" here means the default weight provided by the OS font at that size. Bold is available via `pygame.font.SysFont(None, size, bold=True)` if needed for active player highlight (see Color section).

Line height: single-line renders only (no multiline wrapping in this phase). Vertical stacking gap between panel lines = 20px (sm=8 padding + 12px font metrics).

Source: CONTEXT.md (Claude's Discretion — pygame default font), RESEARCH.md (Pattern 4)

## Board Geometry

| Parameter | Value | Source |
|-----------|-------|--------|
| Diamond center | `(427, 360)` | Board centered in left 2/3 of 1280×720 window |
| Half-radius R | 280 px | RESEARCH.md Pattern 3 verified |
| Tile half-size s | 24 px | Derived: `R * sqrt(2) / 8 ≈ 49.5px` spacing, `s = spacing / 2` |
| Tile shape | 4-point rhombus via `pygame.draw.polygon` | CONTEXT.md D-03 |
| Corner positions | pos 1 = bottom, pos 9 = right, pos 17 = top, pos 25 = left | CONTEXT.md D-02 + RESEARCH.md geometry note |
| Traversal | Clockwise 1→2→…→32→1 | CONTEXT.md D-02 |
| Tile outline width | 1 px | `pygame.draw.polygon(..., width=1)` |
| Tile fill | Base color from color group or special type | See Color section |

### Tile Label Format

Each tile renders two text lines inside its rhombus centre:

- Line 1: position number (e.g. `"12"`)
- Line 2: space type abbreviation

| SpaceId | Abbreviation |
|---------|-------------|
| 1 (FESTIVAL) | `LE` |
| 2 (CHANCE) | `RUT` |
| 3 (CITY) | `DAT` |
| 4 (GAME) | `MG` |
| 5 (PRISON) | `TU` |
| 6 (RESORT) | `NGHI` |
| 7 (START) | `XP` |
| 8 (TAX) | `THUE` |
| 9 (TRAVEL) | `DL` |
| 10 (GOD) | `MAY` |
| 40 (WATER_SLIDE) | `TRUOT` |

Source: CONTEXT.md D-04, `main.py` `_SPACE_LABELS`

### Player Token Rendering

- Token = single uppercase letter (`A`, `B`, `C`, `D`)
- Rendered at tile centre using `pygame.font.SysFont(None, 16)`
- Color: white `(255, 255, 255)` with 1px dark outline for contrast
- If 2 players share a tile: offset tokens by `±4px` on the x-axis
- If 3 players: triangular offset `(−4, +4)`, `(+4, +4)`, `(0, −4)`
- If 4 players: quad offset `(−4, −4)`, `(+4, −4)`, `(−4, +4)`, `(+4, +4)`
- Bankrupt players: do not render token on board

Source: CONTEXT.md D-05

## Info Panel Layout

Panel occupies `x: 853–1280, y: 0–720` (427 × 720 px).

### Speed Indicator (top of panel)

- Height: 28 px
- Background: `(50, 50, 50)`
- Text: `[PAUSED]` / `[1x]` / `[5x]` / `[MAX]` — centered in 28px row
- Font: `pygame.font.SysFont(None, 22)`
- Text color: `(200, 80, 80)` when PAUSED, `(180, 220, 180)` otherwise

### Player Info Section

- Starts at y: 28 (below speed indicator)
- Each player block height: `(720 - 28 - 180) / num_players` — expands to fill available space
  - 2 players: ~128 px each
  - 3 players: ~85 px each
  - 4 players: ~64 px each
- Each block contains 3 lines (from CONTEXT.md D-07):
  - Line 1: Player letter + name (e.g. `A — Nguoi choi A`) — font size 22, highlighted if active
  - Line 2: Cash: `Tien: $1,000,000` — font size 18
  - Line 3: Total assets: `Tai san: $1,000,000` — font size 18
- Active player block: background rect `(60, 80, 110)`
- Bankrupt player: all 3 lines greyed out `(120, 120, 120)`, append `[PHA SAN]` in `(200, 80, 80)` after line 1
- Separator: 1px line `(80, 80, 80)` between player blocks

### Event Log Section

- Position: bottom 180 px of panel (`y: 540–720`)
- Top border: 1px line `(80, 80, 80)` separating from player info
- Shows 8–10 most recent event lines (uses `collections.deque(maxlen=10)`)
- Auto-scroll: newest entry at bottom, oldest falls off top
- Font: `pygame.font.SysFont(None, 18)`
- Text color: `(180, 180, 180)` for regular lines
- Background: `(35, 35, 35)`
- Line height: 16 px (fits 10 lines in 180px with 4px padding top/bottom)
- Long lines truncate at panel edge (no word wrap in this phase)

Source: CONTEXT.md D-06–D-09

## Speed Control

| Speed Level | Delay Between step() | Display Text | Keyboard |
|------------|----------------------|--------------|---------|
| PAUSED | background thread blocked on `threading.Event.wait()` | `[PAUSED]` | `Space` (toggle) |
| 1x | 800 ms (`time.sleep(0.8)`) | `[1x]` | `1` key |
| 5x | 160 ms (`time.sleep(0.16)`) | `[5x]` | `2` key |
| Max | no delay (0 ms) | `[MAX]` | `3` key |

- `Space` key toggles between PAUSED and the last active speed (resumes at 1x if coming from Pause)
- Speed change is immediate (background thread reads `_speed` on next iteration)

Source: CONTEXT.md D-13, D-14, D-15

## Interaction Model

| Input | Action | Effect |
|-------|--------|--------|
| `Space` | Toggle Pause/Resume | PAUSED → previous speed; running → PAUSED |
| `1` | Set speed 1x | 800ms delay; updates speed indicator |
| `2` | Set speed 5x | 160ms delay; updates speed indicator |
| `3` | Set speed Max | No delay; updates speed indicator |
| Window close (X button) | `pygame.QUIT` event | Stop background thread (daemon); call `pygame.quit()`, exit |
| Any other key | Ignored | No action |

No mouse interaction in this phase (all keyboard-only as per CONTEXT.md D-14).

Source: CONTEXT.md D-13, D-14

## Threading and State Contract

| Concern | Constraint |
|---------|-----------|
| Render loop thread | Main thread only — all `pygame.*` calls stay here |
| Game logic thread | Background daemon thread via `threading.Thread(daemon=True)` |
| Shared state access | Protected by single `threading.Lock` |
| UI state shape | `{player_id: {cash, total_assets, position, is_bankrupt}}` + `board_ownership: {position: owner_id}` + `event_log: deque(maxlen=10)` + `active_player_id: str` |
| When to subscribe | EventBus subscriptions registered BEFORE background thread starts |
| total_assets computation | Computed inside EventBus handler at event time (reads Board under background thread context); cached result stored in UI state dict; render loop reads cached value only |

Source: CONTEXT.md D-10–D-12, RESEARCH.md Pitfall 2 + Pitfall 4

## Out of Scope (Deferred to Phase 4)

- End-game stats screen

- Tile artwork, background images, or per-tile icons
- Full VIZ-01–VIZ-06 spec requirements
